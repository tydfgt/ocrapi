系统架构

系统采用端—边—云三层架构。ESP32-CAM 作为采集终端部署于焊机表头，通过 WiFi 将实时图像上传至 Jetson Orin Nano 边缘计算节点；边缘节点运行 PaddleOCR C++ 推理引擎完成文字检测与识别，经由 FastAPI 将结构化结果以 JSON 格式返回。Web 管理面板提供实时硬件监控（GPU 利用率、温度、功耗）与 OCR 历史记录可视化。

研发工作量明细

01
C++ 推理引擎部署与调试
在 ARM aarch64 架构上完成 Paddle Inference 库的跨版本兼容适配（CUDA 11.4 → 12.6），解决 MKL/MKLDNN 不可用、模型名称不匹配、编译 OOM 等 10 余项技术问题。
C++ · Paddle Inference · CUDA
02
TensorRT 推理支持
修改 PaddleOCR C++ 源码 5 个文件，新增 paddle_trt_fp16 / paddle_trt_fp32 运行模式，配置动态 batch 与 1GB workspace，完成 TRT 引擎构建与对比基准测试。
TensorRT · 源码修改 · 性能优化
03
4 模式性能基准测试
对 Paddle FP32、Paddle FP16、TRT FP16、CPU 四种推理模式进行系统基准测试，记录耗时、GPU 利用率、显存占用，形成量化对比报告。
Benchmark · 性能分析
04
FastAPI 服务层开发
编写 310 行 server.py，实现 lifespan GPU 预热、OCR 文件上传 / Base64 双模式接口、硬件状态实时查询、CORS 跨域支持、异步并发处理。
