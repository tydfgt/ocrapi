## 1. 软件概述

### 1.1 功能介绍

PaddleOCR 智能文字识别系统是一套运行于 NVIDIA Jetson Orin Nano 边缘计算平台上的实时 OCR（光学字符识别）服务软件。本软件利用深度学习技术，对用户上传的图片进行文字检测与识别，返回图片中所有文字的内容及置信度。

**核心功能**：
- **图片文字识别**：支持 JPG、PNG、BMP 格式图片上传，自动检测并识别图片中的文字
- **Web 可视化前端**：提供浏览器端交互界面，支持拖拽上传、结果列表展示和可视化标注
- **RESTful API 服务**：提供标准 HTTP 接口，便于集成到其他业务系统
- **Base64 图片识别**：支持通过 JSON 请求体发送 Base64 编码的图片数据
- **硬件实时监控**：前端面板实时显示 GPU 使用率、CPU 使用率、内存占用、温度、功耗等
- **GPU 预热加速**：服务启动时自动加载模型到 GPU，避免首次请求的冷启动延迟

### 1.2 技术特点

| 特点 | 说明 |
|------|------|
| 边缘推理 | 全部计算在 Jetson Orin Nano 本地完成，无需网络，数据安全 |
| GPU 加速 | 利用 CUDA 12.6 + cuDNN 9.3 进行 GPU 并行计算加速 |
| 低延迟 | 预热后每张图片推理约 5.5 秒（30 个文本区域） |
| 轻量级 | PP-OCRv5 Mobile 模型（检测 4.8MB + 识别 17MB） |
| 跨平台 API | 标准 RESTful 接口，任何语言均可调用 |

## 3. 安装与部署

### 3.1 获取源代码

```bash
# 创建工作目录
mkdir -p ~/ocrcplus && cd ~/ocrcplus

# 从 Gitee 克隆 PaddleOCR（国内更快）
git clone --depth=1 https://gitee.com/paddlepaddle/PaddleOCR.git
```

### 3.2 下载 Paddle Inference 推理库

```bash
cd ~/ocrcplus

# 下载 Jetson Orin 专用预编译推理库（约 414MB）
wget -O paddle_inference_install_dir.tgz \
  "https://paddle-inference-lib.bj.bcebos.com/3.0.0/cxx_c/Jetson/jetpack5.1.2_gcc9.4/orin/paddle_inference_install_dir.tgz"

# 解压
tar xzf paddle_inference_install_dir.tgz
```

![推理库下载](../screenshots/02-inference-lib.png)

### 3.3 编译 C++ OCR 引擎

```bash
cd ~/ocrcplus/PaddleOCR/deploy/cpp_infer
mkdir build && cd build

# CMake 配置
cmake .. \
  -DPADDLE_LIB=~/ocrcplus/paddle_inference_install_dir \
  -DOPENCV_DIR=/usr/local \
  -DCUDA_LIB=/usr/local/cuda-12.6/lib64 \
  -DCUDNN_LIB=/usr/lib/aarch64-linux-gnu \
  -DWITH_MKL=OFF \
  -DWITH_GPU=ON \
  -DWITH_STATIC_LIB=OFF \
  -DUSE_FREETYPE=OFF

# 编译（-j2 防止内存溢出）
make -j2
```

编译完成后生成 `ppocr` 可执行文件（约 56MB）：

```bash
ls -lh ./ppocr
# -rwxrwxr-x  56M  ppocr

file ./ppocr
# ELF 64-bit LSB pie executable, ARM aarch64, dynamically linked
```

![编译完成](../screenshots/04-compile.png)

### 3.4 安装 Python 依赖

```bash
cd ~/ocrcplus

# 创建虚拟环境
python3 -m venv venv

# 激活虚拟环境
source venv/bin/activate

# 安装依赖
pip install fastapi uvicorn python-multipart Pillow psutil jetson-stats aiofiles aiohttp
```

### 3.5 下载 OCR 模型

```bash
cd ~/ocrcplus && mkdir -p models && cd models

# 文本检测模型（4.8MB）
wget "https://paddle-model-ecology.bj.bcebos.com/paddlex/official_inference_model/paddle3.0.0/PP-OCRv5_mobile_det_infer.tar"
tar xf PP-OCRv5_mobile_det_infer.tar

# 文本识别模型（17MB）
wget "https://paddle-model-ecology.bj.bcebos.com/paddlex/official_inference_model/paddle3.0.0/PP-OCRv5_mobile_rec_infer.tar"
tar xf PP-OCRv5_mobile_rec_infer.tar
```

### 3.6 启动服务

```bash
cd ~/ocrcplus
./start.sh
```

启动日志：

## 4. Web 前端使用指南

### 4.1 访问前端

在浏览器地址栏输入 `http://{Jetson设备IP}:8899`（例如 `http://192.168.3.110:8899`）。

### 4.2 页面布局说明

| 区域 | 位置 | 功能 |
|------|------|------|
| 标题栏 | 顶部 | 显示软件名称、设备型号、模型版本 |
| 硬件监控面板 | 标题栏下方 | 实时显示 GPU、CPU、内存、温度、功耗、磁盘 |
| 上传区域 | 中部 | 点击/拖拽/粘贴图片进行 OCR |
| 识别结果区 | 左侧 | 列出检测到的所有文字及置信度 |
| 可视化区 | 右侧 | 显示标注了检测框和识别文字的图片 |
| API 信息条 | 底部 | 显示 L4T/CUDA/cuDNN 等版本信息 |

### 4.3 上传图片

支持三种上传方式：

**方式一：点击上传**
1. 点击上传区域（"点击 / 拖拽 / Ctrl+V 粘贴图片"）
2. 在弹出的文件选择对话框中选取图片文件
3. 系统自动开始识别

**方式二：拖拽上传**
1. 将图片文件从文件管理器拖拽到上传区域
2. 松开鼠标，系统自动开始识别

**方式三：剪贴板粘贴**
1. 复制一张图片到剪贴板（如截图后 Ctrl+C）
2. 在页面任意位置按 Ctrl+V
3. 系统自动开始识别

### 4.4 查看识别结果

上传图片后，页面会显示：

- **识别结果列表**：左侧面板列出每个检测到的文字，后面跟着置信度分数
  - 🟢 绿色：置信度 > 0.9（高置信度）
  - 🟡 黄色：置信度 0.7~0.9（中等置信度）
  - 🔴 红色：置信度 < 0.7（低置信度）
- **可视化图片**：右侧面板显示标注了检测框（绿色边框）和识别文字的图片

![OCR 结果](../screenshots/03-ocr-result.png)

### 4.5 硬件监控面板

页面顶部的硬件监控面板每 5 秒自动刷新一次，显示以下实时数据：

| 指标 | 数据来源 | 说明 |
|------|----------|------|
| GPU | jtop | GPU 利用率百分比 |
| CPU | jtop | 6 核 CPU 平均利用率 |
| 内存 | jtop | 已用/总量（GB） |
| GPU 温度 | jtop | GPU 核心温度（°C） |
| 功耗 | jtop | 整机功耗（W） |
| 磁盘 | psutil | 已用/总量（GB） |

## 5. API 接口使用指南

### 5.1 接口概览

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/` | Web 前端页面 |
| POST | `/ocr` | 上传图片进行 OCR 识别 |
| POST | `/ocr_base64` | Base64 编码图片 OCR |
| GET | `/status` | 机器实时状态 |
| GET | `/health` | 健康检查 |
| GET | `/docs` | Swagger API 交互文档 |

### 5.2 POST /ocr — 图片文字识别

**请求格式**：`multipart/form-data`

**参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| file | File | 是 | 图片文件，支持 JPG/PNG/BMP |

**curl 调用示例**：

```bash
curl -X POST http://192.168.3.110:8899/ocr -F "file=@test.png"
```

**Python 调用示例**：

```python
import requests

with open("test.png", "rb") as f:
    resp = requests.post("http://192.168.3.110:8899/ocr", files={"file": f})
    data = resp.json()
    for text, score in zip(data["texts"], data["scores"]):
        print(f"[{score:.2f}] {text}")
```

**响应格式**：

```json
{
  "texts": ["登机牌", "BOARDINGPASS", "航班FLIGHT", ...],
  "scores": [0.996, 0.993, 0.997, ...],
  "count": 30,
  "time_s": 5.84,
  "vis_base64": "iVBORw0KGgo...",
  "json": { /* 完整原始结果 */ }
}
```

**响应字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| texts | string[] | 识别出的文字列表 |
| scores | float[] | 每个文字的置信度（0~1） |
| count | int | 检测到的文本区域数量 |
| time_s | float | 推理耗时（秒） |
| vis_base64 | string | 标注图片的 Base64 编码 |
| json | object | 完整原始检测+识别结果 |

![API 文档](../screenshots/02-api-docs.png)

## 6. 命令行使用指南

### 6.1 直接调用 C++ 引擎

除 Web API 外，也可以直接调用 C++ 推理引擎进行 OCR：

```bash
cd ~/ocrcplus

./PaddleOCR/deploy/cpp_infer/build/ppocr ocr \
  --input ./models/test.png \
  --text_detection_model_dir ./models/PP-OCRv5_mobile_det_infer \
  --text_detection_model_name PP-OCRv5_mobile_det \
  --text_recognition_model_dir ./models/PP-OCRv5_mobile_rec_infer \
  --text_recognition_model_name PP-OCRv5_mobile_rec \
  --use_doc_orientation_classify false \
  --use_doc_unwarping false \
  --use_textline_orientation false \
  --device gpu \
  --save_path ./output/
```

### 6.2 命令行参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--input` | 输入图片路径 | 必填 |
| `--device` | 推理设备（gpu/cpu） | gpu |
| `--precision` | 精度（fp32/fp16/trt_fp32/trt_fp16） | fp32 |
| `--text_detection_model_dir` | 检测模型目录 | 必填 |
| `--text_recognition_model_dir` | 识别模型目录 | 必填 |
| `--save_path` | 结果保存路径 | ./output/ |

### 6.3 支持的精度模式

| 模式 | `--precision` 值 | 说明 |
|------|-----------------|------|
| 原生 FP32 | `fp32` | 默认模式，速度最快 |
| 原生 FP16 | `fp16` | 半精度（本设备不推荐） |
| TensorRT FP32 | `trt_fp32` | TRT 加速 FP32 |
| TensorRT FP16 | `trt_fp16` | TRT 加速 FP16 |

## 8. 常见问题与故障排除

### Q1: 启动时提示 "jtop 状态采集已启动" 但前端显示 "--"

**原因**：jtop 需要约 2 秒初始化，前端每 5 秒刷新。  
**解决**：等待 5-10 秒后刷新页面即可。

### Q2: CPU 模式下推理崩溃（Segmentation fault）

**原因**：ARM 平台不支持 Intel MKL/MKLDNN 库。  
**解决**：使用 `--device gpu` 参数。

### Q3: 编译时内存不足（OOM）

**原因**：make -j4 或 -j6 并发数过高。  
**解决**：使用 `make -j2` 或 `make -j1`，确保 Swap 已启用。

### Q4: "Model name mismatch" 错误

**原因**：模型名称参数与实际模型目录不匹配。  
**解决**：确保 `--text_detection_model_name` 与模型目录名一致。mobile 模型用 `PP-OCRv5_mobile_det`，server 模型用 `PP-OCRv5_server_det`。

### Q5: OCR 识别结果不理想

**建议**：
- 确保图片清晰，文字与背景对比度高
- 尝试使用 server 版模型（精度更高但速度较慢）
- 调整图片分辨率，建议文字高度 > 20 像素
