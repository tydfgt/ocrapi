## 1. 引言

### 1.1 编写目的

本文档为 **PaddleOCR 智能文字识别系统 V1.0** 的软件设计说明书，详细描述系统架构、模块划分、接口规范、核心算法流程及部署方案，供著作权审查及后续维护参考。

### 1.2 项目背景

本系统是一套基于 NVIDIA Jetson Orin Nano 边缘计算平台的**实时 OCR 文字识别服务**。利用 PaddlePaddle 深度学习框架的 C++ 推理引擎，结合 PP-OCRv5 模型，在 ARM 架构的边缘设备上实现高效、低延迟的图片文字检测与识别。

**核心特点**：
- 边缘端 GPU 加速推理
- RESTful API 服务化部署
- Web 前端可视化交互
- 实时硬件监控

### 1.3 定义与缩写

| 缩写/术语 | 全称/说明 |
|-----------|-----------|
| OCR | Optical Character Recognition，光学字符识别 |
| PP-OCRv5 | PaddlePaddle OCR 第5代模型 |
| Paddle Inference | 飞桨深度学习推理引擎 |
| CUDA | NVIDIA 并行计算平台 |
| cuDNN | NVIDIA 深度神经网络加速库 |
| TensorRT | NVIDIA 推理优化引擎 |
| L4T | Linux for Tegra，NVIDIA Jetson 操作系统 |
| FastAPI | Python 高性能 Web 框架 |
| jtop | Jetson 硬件状态监控库 |

### 2.3 构建流程

#### 2.3.1 获取源码与依赖

```bash
# 1. 创建工作目录
mkdir -p ~/ocrcplus && cd ~/ocrcplus

# 2. 克隆 PaddleOCR 源码 (从 Gitee, 国内更快)
git clone --depth=1 https://gitee.com/paddlepaddle/PaddleOCR.git
# 输出: Cloning into 'PaddleOCR'... done.

# 3. 下载 Paddle Inference 预编译推理库 (Jetson Orin 专用, ~414MB)
wget "https://paddle-inference-lib.bj.bcebos.com/3.0.0/cxx_c/Jetson/jetpack5.1.2_gcc9.4/orin/paddle_inference_install_dir.tgz"
# 输出: paddle_inference_install_dir.tgz  saved [414M]

# 4. 解压推理库
tar xzf paddle_inference_install_dir.tgz

# 5. 查看推理库版本信息
cat paddle_inference_install_dir/version.txt
```

#### 2.3.2 编译 C++ 推理引擎

进入源码目录并创建构建目录：

```bash
cd PaddleOCR/deploy/cpp_infer
mkdir build && cd build
```

**CMake 配置** — 生成构建文件：

```bash
cmake .. \
  -DPADDLE_LIB=~/ocrcplus/paddle_inference_install_dir \
  -DOPENCV_DIR=/usr/local \
  -DCUDA_LIB=/usr/local/cuda-12.6/lib64 \
  -DCUDNN_LIB=/usr/lib/aarch64-linux-gnu \
  -DWITH_MKL=OFF \          # ARM 平台不支持 Intel MKL
  -DWITH_GPU=ON \           # 启用 GPU 加速
  -DWITH_STATIC_LIB=OFF \   # 使用动态链接库
  -DUSE_FREETYPE=OFF        # 不启用 FreeType 字体渲染
```

### 3.2 模块划分

| 模块 | 文件 | 功能 | 技术 |
|------|------|------|------|
| Web 前端 | `static/index.html` | 上传/展示/硬件监控 | HTML5 + Fetch |

### 3.3 数据流

```

    → 文本检测 (DB算法) → 文本识别 (CRNN) → JSON + PNG 输出
    → FastAPI 读取结果 → Base64 编码 → HTTP JSON 返回 → 前端渲染
```

## 4. 模块详细设计

### 4.1 API 服务模块

**生命周期**：

```
启动 → jtop线程 → 预热推理(GPU) → 接受请求 → 关闭
       (每秒采集)  (加载模型)      (运行中)
```

**核心函数**：

| 函数 | 说明 |
|------|------|
| `run_ocr(path)` | 调用 ppocr 子进程执行 OCR |
| `_jtop_thread()` | 后台线程，每 1 秒采集硬件状态 |
| `ocr(file)` | POST /ocr 处理，返回 JSON |
| `status()` | GET /status，返回实时硬件数据 |

**硬件状态数据结构**：
```json

    "gpu_percent": 0.0,
    "temps_c": {"cpu":44.2,"gpu":46.1},
    "power_mw": {"TOT":4607},
    "nvp_model": "MAXN_SUPER"
  }
}
```

### 4.2 C++ OCR 引擎 (ppocr)

**源码结构**：

```
ppocr
├── cli.cc                         # 命令行入口
├── src/base/
│   ├── base_predictor.cc          # run_mode 映射逻辑
│   └── base_pipeline.cc           # 流水线基类
├── src/common/
│   └── static_infer.cc            # Paddle Config 配置
│       ├── GPU: EnableUseGpu()
│       ├── TRT: EnableTensorRtEngine()  ← 新增
│       └── FP16: kHalf Precision
├── src/modules/
│   ├── text_detection/            # DB 文本检测
│   └── text_recognition/          # CRNN 文本识别
├── src/pipelines/ocr/             # 检测→识别 流水线
└── src/utils/
    ├── pp_option.h                # 运行模式定义(含新增TRT)
    ├── args.cc                    # 命令行参数
    └── utility.cc                 # 工具函数
```

**支持的运行模式**：

| 模式 | 标识 |
|------|------|
| 原生 FP32 | `paddle` |
| 原生 FP16 | `paddle_fp16` |
| TensorRT FP32 | `paddle_trt_fp32`（新增） |
| TensorRT FP16 | `paddle_trt_fp16`（新增） |

**源码修改说明**（支持 TensorRT）：

`pp_option.h` — 新增运行模式：
```cpp
const std::vector<std::string> SUPPORT_RUN_MODE = {
    "paddle", "paddle_fp16",
    "paddle_trt_fp32", "paddle_trt_fp16",  // 新增
    "mkldnn", "mkldnn_bf16"
};
```

`static_infer.cc` — TensorRT Engine 配置：
```cpp
if (option_.RunMode().find("paddle_trt") != std::string::npos) {
    config.EnableTensorRtEngine(
        1 << 30,    // 1GB workspace
        -1,         // 动态 batch
        2,          // min subgraph size
        trt_precision,
        true,       // 缓存序列化 engine
        false);     // 非 INT8 校准
    config.EnableTunedTensorRtDynamicShape();
}
```

## 5. 接口设计

### 5.1 POST /ocr — 图片文字识别

- Content-Type: `multipart/form-data`
- 参数: `file` (图片文件)
- 响应: `{"texts":[...], "scores":[...], "count":N, "time_s":f, "vis_base64":"..."}`

### 5.2 POST /ocr_base64 — Base64 图片识别

- Content-Type: `application/json`
- 参数: `{"image": "base64字符串"}`
- 响应: 同上

### 5.3 GET /status — 机器实时状态

- 响应: `{"device":{...}, "realtime":{...}, "api":{...}}`

### 5.4 GET / — 前端页面

返回 HTML 交互界面。

### 5.5 GET /docs — Swagger API 文档

FastAPI 自动生成。

## 6. 核心算法与流程

### 6.1 OCR 推理流程

```
输入图片 → OpenCV解码 → 文本检测(DB算法, GPU)
    → N个文本区域坐标 → 文本识别(CRNN算法, GPU×N)
    → 文字+置信度 → JSON序列化 + 可视化绘图 → 输出
```

### 6.2 GPU 预热机制

```
服务启动 → jtop 后台线程 → 运行一次 OCR 预热推理
    → 加载模型到 GPU 显存 → 初始化 CUDA 上下文 → 预热 cuDNN cache
    → GPU 进入热状态 (冷启动 ~8s → 预热后 ~5.5s)
```

### 6.3 性能数据

| 指标 | 值 |
|------|-----|
| 模型 | PP-OCRv5 Mobile (检测 4.8MB + 识别 17MB) |
| 测试图片 | 126KB PNG (登机牌, 30 文本区域) |
| 热推理 | ~5.55s/图 |
| GPU 使用率 | 推理时 ~85-95% |
