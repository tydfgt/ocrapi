# 源代码文档说明

---

## 一、源代码提交规范

### 1.1 总体要求

- 提交的源程序需为**完整代码**，具有原始性，不可包含自动生成代码
- 总共提交 **60 页** 源代码（前 30 页 + 后 30 页）
- 每页代码不少于 **50 行**（建议设置为 55 行/页）
- 页眉标注软件名称及版本号，页脚标注页码

### 1.2 格式要求

| 项目 | 要求 |
|------|------|
| 纸张大小 | A4 |
| 页边距 | 上下 2.54cm，左右 3.18cm |
| 字体 | 等宽字体（如 Consolas、Courier New、Source Code Pro） |
| 字号 | 小五号或五号（9pt - 10.5pt） |
| 行间距 | 单倍行距 |
| 页眉 | 软件全称 + 版本号（居中） |
| 页脚 | 第 X 页 / 共 60 页（居中） |

### 1.3 内容要求

- **前 30 页**：从程序开头选取，一般为软件的主入口、核心初始化代码
- **后 30 页**：从程序末尾选取，一般为工具函数、配置、底层实现代码
- 代码应体现**原创性**和**完整性**，避免全篇注释或无意义空行
- 可删除部分非核心、重复性代码，但不能影响代码逻辑的完整呈现

## 二、源代码整理方法

### 2.1 推荐整理流程

1. **汇总源文件**：将项目所有手写源代码文件整理到一个目录
2. **去除冗余**：排除第三方库、node_modules、自动生成代码等
3. **排序**：按代码重要性/层级由核心到外围排序
4. **拼接**：将所有源文件拼接为一个文档
5. **分页**：设置页码格式，确保每页 50+ 行
6. **标记**：在每段文件开头添加注释标明文件名，例如：`// ===== src/main/App.java =====`

### 2.2 不纳入源代码的内容

- 第三方开源库代码（如 jQuery、Bootstrap 等）
- 自动生成代码（如 protobuf、gRPC 生成代码、MyBatis 生成器等）
- 框架核心代码（如 Spring Boot 内核、.NET 运行时等）
- node_modules / vendor 等依赖目录
- 编译产物（.class、.dll、.exe、dist/ 等）
- 配置文件中的服务器地址、密钥等敏感信息（可脱敏后保留）

package com.example;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

// ===== src/main/java/com/example/controller/UserController.java =====
package com.example.controller;

import com.example.entity.User;
import com.example.service.UserService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/api/users")
public class UserController {

    @Autowired
    private UserService userService;

    @GetMapping
    public List<User> list() {
        return userService.findAll();
    }

    @GetMapping("/{id}")
    public User getById(@PathVariable Long id) {
        return userService.findById(id);
    }

    @PostMapping
    public User create(@RequestBody User user) {
        return userService.save(user);
    }

    @PutMapping("/{id}")
    public User update(@PathVariable Long id, @RequestBody User user) {
        user.setId(id);
        return userService.update(user);
    }

    @DeleteMapping("/{id}")
    public void delete(@PathVariable Long id) {
        userService.delete(id);
    }
}

## 四、提取源代码脚本（辅助工具）

以下脚本可以帮助自动提取并格式化源代码（以 Python 为例）：

```python
import os
import glob

# 配置
PROJECT_DIR = "/path/to/your/project"
OUTPUT_FILE = "源代码文档.txt"
EXTENSIONS = ["*.java", "*.kt", "*.py", "*.js", "*.ts", "*.vue", "*.cs"]
EXCLUDE_DIRS = ["node_modules", "vendor", ".git", "dist", "build", "target", "__pycache__"]

def collect_sources():
    files = []
    for ext in EXTENSIONS:
        for filepath in glob.glob(f"{PROJECT_DIR}/**/{ext}", recursive=True):
            # 排除特定目录
            if not any(ex in filepath for ex in EXCLUDE_DIRS):
                files.append(filepath)
    return sorted(files)

def generate_document(files):
    with open(OUTPUT_FILE, "w", encoding="utf-8") as out:
        for filepath in files:
            rel_path = os.path.relpath(filepath, PROJECT_DIR)
            out.write(f"\n// ===== {rel_path} =====\n\n")
            with open(filepath, "r", encoding="utf-8", errors="ignore") as f:
                out.write(f.read())
            out.write("\n")
    print(f"源代码已整理到：{OUTPUT_FILE}")
    print(f"共 {len(files)} 个文件")

if __name__ == "__main__":
    files = collect_sources()
    generate_document(files)
```
